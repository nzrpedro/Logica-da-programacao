Pesquisa sobre branches GitHub

As Branches são ramificações de um projeto feitas pelos desenvolvedores no Git. Elas servem para criar uma versão separada do código principal, 
assim é possível mexer em alguma coisa sem estragar o que já está funcionando.
No dia a dia, as branches ajudam a organizar o trabalho, deixar tudo mais seguro e permitir que várias pessoas trabalhem ao mesmo tempo no mesmo projeto sem confusão.
Um exemplo prático é quando o sistema de login do projeto apresenta um erro. Nesse caso, é criada uma branch só para corrigir esse problema.
Dessa forma, os usuários continuam usando o sistema normalmente enquanto a correção é feita. Outro exemplo é quando se quer adicionar uma nova função, como uma página de cadastro de usuários.
A equipe pode criar uma branch só para essa funcionalidade até ela estar pronta. Um terceiro exemplo é para testes e experimentos.
Se alguém quer testar uma nova ideia ou interface, pode fazer isso em uma branch separada.
Se não funcionar, a branch pode ser descartada sem prejudicar o projeto principal.
As branches são fundamentais porque deixam o desenvolvimento mais organizado, seguro e permitem que o projeto cresça sem causar problemas para quem já está usando a versão principal.
Por isso, qualquer desenvolvedor que trabalha com Git precisa saber criar e usar branches corretamente.
Com elas o trabalho entre desenvolvedores se torna muito mais simples e coordenado de forma que todos tenham uma ideia e sejam incluidos em novas durante o processo de desenvolvimento do programa.
Se um desenvolvedor desconhece o funcionamento de branches, a quantidade de vagas que ele pode cumprir se torna muito menor, dificultando o processo de encontrar empregos, mesmo que no ramo freelancer.
Caso algum participante de um projeto que participe diretamente do desenvolvimento desconheça ou não saiba o suficiente da criação de branches, 
pode acabar atrapalhando completamente a organização, cooperação e funcionamento do código/programa.
